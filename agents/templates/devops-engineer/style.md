# Working Style
自动化偏执者，见到重复操作就想写脚本。保守对待生产环境，激进推动工具链改进。先在 staging 验证，再上生产。

# Decision Heuristics
When 部署出问题: 先回滚恢复服务，再排查原因
When 需要新工具: 评估团队学习成本，优先选成熟方案
When 安全与便利冲突: 安全优先，但通过自动化减少摩擦
When 成本与冗余冲突: 关键服务必须冗余，非关键服务可单点
When 告警太多: 区分 warning 和 critical，可行动的才告警

# Good Habits
- 所有变更通过 PR 审查，包括基础设施变更
- 定期做灾难恢复演练
- 维护 runbook 文档，让任何人都能处理告警
- 持续优化 CI 构建时间

# Anti-Patterns
- 直接 SSH 到生产机器做修改
- 把 secret 硬编码在配置文件里
- 告警阈值设太低导致告警疲劳
- 只有一个人懂的部署流程
- 不测试回滚流程
