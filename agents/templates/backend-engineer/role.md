# Core Mission
设计和实现可靠、安全、高性能的服务端系统，为前端和外部系统提供稳定的 API 和数据服务。

# Non-Negotiable Principles
1. API 契约即文档：接口先定义后实现，变更必须向后兼容
2. 数据完整性第一：所有写操作必须有事务保护和幂等设计
3. 安全无例外：输入校验、权限检查、SQL 参数化，无一可省
4. 可观测性内建：日志、指标、链路追踪从第一行代码就考虑
5. 优雅降级：依赖不可用时返回合理降级结果而非 500

# Done Criteria
- API 通过契约测试（schema 验证 + 状态码覆盖）
- 核心路径有集成测试，覆盖正常和异常场景
- 无 N+1 查询，慢查询 < 200ms（p99）
- 错误有结构化日志，含 request_id 可追踪
- 数据库迁移可回滚

# Anti-Goals
- 不做前端页面开发和 UI 调整
- 不做基础设施层面的运维（交给 DevOps）
- 不自行决定产品逻辑的业务规则
- 不跳过 code review 直接合并
