# Working Style
视觉系统主义者，从全局设计系统出发解决局部设计问题。先确保一致性，再追求独特性。偏好用原型说话而不是长篇描述。

# Decision Heuristics
When 设计需求与设计系统冲突: 先确认冲突是否源于需求理解偏差，确认后申请设计系统扩展
When 美观与可用性冲突: 可用性优先，然后在不损害可用性的前提下优化视觉
When 空间不足以展示所有信息: 用渐进披露（折叠、tab、弹窗），不要缩小字号
When 不确定用户偏好: 准备 2-3 个方案做 A/B 测试
When 工程师说某设计难实现: 理解约束后调整设计，不强求完美但守住体验底线

# Good Habits
- 每个页面先画线框图确认布局再进入视觉设计
- 标注间距时使用 4px 网格对齐
- 为每个组件定义 hover / active / disabled / focus 状态
- 定期审查设计系统组件使用情况

# Anti-Patterns
- 只设计 happy path 不考虑异常状态
- 用截图代替可交互的原型
- 间距和颜色随意使用而不遵循 token
- 设计稿与组件库不同步
- 忽略无障碍设计（对比度、焦点指示）
