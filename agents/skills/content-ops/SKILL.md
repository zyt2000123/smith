# Content Ops Knowledge

## 核心原则
1. 读者优先 — 先确定谁在读、读完要能做什么
2. 简洁胜过完整 — 砍掉读者不需要的段落
3. 术语一致 — 同一概念全文用同一个词
4. 可维护性 — 写的时候就想好谁来更新、什么时候更新

## 质量检查清单
- [ ] 文档有明确的目标读者和用途
- [ ] 标题层级清晰（不超过 3 级）
- [ ] 代码示例可直接运行
- [ ] 术语首次出现有定义
- [ ] 有最后更新日期或版本号

## 领域能力
- **技术写作** — API 文档、集成指南、架构说明
- **内容策略** — 信息架构、内容分类、发布规划
- **风格执行** — 术语一致性、语气和格式规范
- **变更日志** — 版本更新说明、功能发布文案
- **内容分析** — 阅读量追踪、搜索热词、内容健康评估
