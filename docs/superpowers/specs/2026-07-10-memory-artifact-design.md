# Memory 设计 Artifact 规格

## 目标

制作一页可直接打开的技术设计 artifact，向引擎开发者、评审 Agent 和面试官解释 Agent-Smith 的 Memory 设计：记忆如何写入、如何按时间编译、如何被 Dream 整理、如何通过被动注入与主动检索进入当前任务，以及为什么这些取舍适合单机常驻 Agent。

## 设计判断

核心命题是：**Memory 不是聊天日志，而是一条被治理的事实生命周期。** 页面必须把这条命题放在首屏，并将设计拆成可扫描的小问题，而不是堆叠实现术语。

## 页面结构

1. **首屏 thesis**：标题、短句、范围标签和四个关键指标。
2. **生命周期总览**：`write → compile → dream → retrieve` 四阶段横向流图；每阶段有输入、输出和一句解释。
3. **四层记忆编译**：`today / week / longterm / facts` 作为四张时间卡片，说明保留什么、多久重建、如何形成画像。
4. **写入与存储**：展示 `recent.jsonl`、`agent/*.md`、`project/*.md`、`search.sqlite` 的关系，并强调 Markdown 是唯一事实源、索引是可丢弃派生物。
5. **Dream 整理**：使用四个操作卡片表达清秘、剪枝、去重合并、模式提取，并呈现 `preview → apply` 的安全边界。
6. **双轨读取**：对比被动注入和主动检索，解释画像摘要与 query-time top-k 如何互补。
7. **检索降级**：从 `FTS5 + vector + RRF` 到纯 FTS5、关键词扫描、空结果的降级阶梯。
8. **设计决策**：用 3 个高信息量 decision cards 解释明文文件、时间分层、Dream 的取舍。
9. **尾屏**：以一段深色伪代码收束，强调 best-effort、token 有界和用户可审计。

## 交互

- 顶部阶段导航按钮可滚动到对应阶段，并同步高亮。
- 生命周期节点可点击，更新右侧“当前阶段”说明面板；小屏幕时该面板下移。
- 不做复杂状态管理；交互只用于帮助阅读，不改变内容事实。

## 视觉系统

- 页面底色：`#F3F5F8`，带轻微蓝灰倾向，避免纯灰的廉价感。
- 主文本：`#17324D`；主路径：`#2D6CDF`。
- 健康/完成：`#DDF3E6`；中间态：`#EDE7F6`；风险：`#F8E6B8`；阻断仅使用 `#C94B4B`。
- 标题与正文均使用系统中文无衬线；数据、文件名和伪代码使用等宽字体。
- 版心最大宽度约 1120px，白卡片统一 1px 边框、10–14px 圆角，依靠垂直节奏而非阴影分隔。
- 图示采用圆角节点、细连接线和低饱和语义色，避免单一巨型图。

## 内容约束

- 所有示例必须来自当前项目真实设计：`had_tools` 门槛、四层编译、30 天未访问剪枝、70% 关键词重叠、top-5 检索、Markdown 事实源等。
- 不能把 `employee_dir` 的兼容命名渲染为产品模型；页面文案使用 Agent / project 语义，并在注释中说明兼容层。
- 明确短期会话历史不属于 Memory 层。
- 不声称当前实现具有超出 SDD 或代码的能力。

## 验收标准

- 文件可通过静态 HTTP 服务打开，桌面和窄屏都无横向溢出。
- 首屏在不滚动时能看懂 Memory 的核心命题和四阶段生命周期。
- 页面至少包含一张生命周期流图、一张四层编译图、一张双轨读取图和一张降级阶梯图。
- 点击生命周期节点会更新说明面板；键盘 focus 可见；`prefers-reduced-motion` 下不依赖动画。
- 所有关键数字、阈值和路径与 `docs/12-记忆系统SDD.md` 保持一致。
