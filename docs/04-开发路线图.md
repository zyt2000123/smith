# 04 · 开发路线图

> 2026-07-03 修订：路线图按当前产品原型重排。v0 不再以圆桌会议为核心，而是先打通「数字员工创建 → 员工主页 → 对话任务 → 自动任务 → 右侧能力面板 → Hub 知识调用」闭环。

## Phase 0 · 技术 spike（1-2 天）

- [x] ~~桌面壳方案对比~~：已确定 SwiftUI 原生 macOS 应用。
- [x] SwiftUI 原型搭建：19 个 Swift 文件，编译通过，覆盖管理页/员工详情/对话/设置。
- [ ] **SwiftUI + FastAPI 联调 spike**：URLSession 调 FastAPI 接口，SSE 事件流在 SwiftUI 中实时渲染。
- [ ] AgentScope 单员工 ReActAgent spike：能接受任务、调用一个 mock 工具、流式返回。
- [ ] Hub 连接 spike：MCP 和直调检索 API 各试一次，确定默认集成方式。
- [ ] 本地目录策略 spike：确认 `~/.agent-smith` 下 employees/tasks/connectors/sqlite 的布局。

**验证点**：SwiftUI 能调通 FastAPI 并渲染 SSE 流；AgentScope 能跑一个单员工任务；Hub 检索链路至少一种方式可用。

## Phase 1 · 本地 server 与数据模型（1-1.5 周）

- [ ] 员工文件模型：`role.md`、`style.md`、`workflow.md`、`config.yaml`、avatar。
- [ ] 员工 CRUD API：创建、列表、详情、编辑、删除。
- [ ] 任务模型：对话任务、自动任务、状态、计划、日志。
- [ ] 会话模型：messages jsonl + SQLite 索引。
- [ ] 连接器模型：Hub MCP/API 配置、token 本地保存。
- [ ] 权限模型：工作目录、工具白名单、询问/允许/拒绝规则。

**验证点**：不用前端，通过 API 完成「创建员工 → 查看详情 → 创建对话任务 → 调用 mock 工具 → 保存工作记录」。

## Phase 2 · SwiftUI 前端接 API（1-1.5 周）

- [ ] SwiftUI 数字员工管理页接真实 API：员工 CRUD、筛选、搜索。
- [ ] 新建员工 Sheet：角色模板、自定义模板、头像、简介。
- [ ] 员工详情首页接 API：身份卡、工作记录、记忆、关于我、能力与工具、原始档案。
- [ ] 对话任务页接 SSE：会话列表、消息流实时渲染、输入区、工作目录选择。
- [ ] 员工子页面接 API：技能、连接器、权限、自动任务、任务列表、记忆。

**验证点**：SwiftUI 前端全部接真实 FastAPI 后端，用户能用原生 UI 完成单员工任务的完整闭环。

## Phase 3 · 任务执行闭环（1-2 周）

- [ ] 对话任务真实执行：AgentScope ReActAgent + SSE 消息流。
- [ ] 计划面板：展示任务步骤、运行状态、失败原因。
- [ ] MCP 面板：展示可用 MCP server，能启用/停用。
- [ ] 技能面板：展示员工可用技能，先支持本地内置技能。
- [ ] 权限面板：敏感工具调用前询问，记录规则。
- [ ] 知识库面板：展示员工可用 Hub 知识范围，支持检索来源回链。

**验证点**：用户能在对话页派任务，右侧面板能同步展示计划、工具、权限、知识调用情况。

## Phase 4 · 自动任务与记忆（1 周）

- [ ] 自动任务创建入口和列表。
- [ ] 定时/手动触发自动任务。
- [ ] 工作记录统计：任务数、项目数、最近活动。
- [ ] 记忆 v0：会话摘要、员工长期记忆、可查看/可清理。
- [ ] 员工主页接真实工作记录和记忆数据。

**验证点**：员工能执行一次自动任务，并在员工主页留下可见工作记录和记忆。

## Phase 5 · 打磨与试点（1 周）

- [ ] 角色模板第一批：前端、后端、产品、运营、测试。
- [ ] 安装包、签名、公证、更新通道。
- [ ] 首启引导：连接 Hub、选择工作目录、创建第一个员工。
- [ ] 试点 2-3 个真实同事，收集任务路径问题。

**验证点**：非开发用户能独立安装、创建员工、连接 Hub、完成第一条对话任务。

## 持续纪律

- 不把能力配置做成散落的设置页，优先落到右侧任务/能力面板。
- 不提前抽象多员工协作，先把单员工生命周期打稳。
- AgentScope 导入收口 adapter。
- SSE/事件契约集中定义。
- Hub 连接封装为 Connector，避免 MCP/HTTP 选择扩散到业务代码。
