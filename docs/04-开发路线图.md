# 04 · 开发路线图

> 2026-07-03 修订：路线图按当前产品原型重排。v0 不再以圆桌会议为核心，而是先打通「Agent创建 → Agent主页 → 对话任务 → 自动任务 → 右侧能力面板 → Hub 知识调用」闭环。

## Phase 0 · 技术 spike（1-2 天）

- [ ] **桌面壳方案对比 spike**：daemon + 浏览器 vs Electron，各做一个最小 PoC（FastAPI 进程随壳启动/退出，前端能访问），对比分发体积、启动速度、托盘/通知能力、打包复杂度。
- [ ] AgentScope 单Agent ReActAgent spike：能接受任务、调用一个 mock 工具、流式返回。
- [ ] Hub 连接 spike：MCP 和直调检索 API 各试一次，确定默认集成方式。
- [ ] 本地目录策略 spike：确认 `~/.agent-smith` 下 employees/tasks/connectors/sqlite 的布局。
- [ ] 前端原型冻结：以当前 `prototype` 为 v0 信息架构基准。

**验证点**：两种壳方案各有可运行 PoC；AgentScope 能跑一个单Agent任务；Hub 检索链路至少一种方式可用。定壳方案后锁定。

## Phase 1 · 本地 server 与数据模型（1-1.5 周）

- [ ] Agent文件模型：`identity.md`、`persona.md`、`bible.md`、`config.yaml`、avatar。
- [ ] Agent CRUD API：创建、列表、详情、编辑、删除。
- [ ] 任务模型：对话任务、自动任务、状态、计划、日志。
- [ ] 会话模型：messages jsonl + SQLite 索引。
- [ ] 连接器模型：Hub MCP/API 配置、token 本地保存。
- [ ] 权限模型：工作目录、工具白名单、询问/允许/拒绝规则。

**验证点**：不用前端，通过 API 完成「创建Agent → 查看详情 → 创建对话任务 → 调用 mock 工具 → 保存工作记录」。

## Phase 2 · 前端主形态（1-1.5 周）

- [ ] 搭正式前端工程：沿用 Hub 的 Vite + React + Tailwind + shadcn/Base UI。
- [ ] Agent管理页：Agent卡片、筛选、搜索、对话任务/自动任务入口。
- [ ] 新建Agent弹窗：角色模板、自定义模板、头像、简介。
- [ ] Agent详情首页：二级菜单、身份卡、工作记录、记忆与积累。
- [ ] 对话任务页：群组、会话列表、任务建议、输入区、工作目录。
- [ ] 右侧任务/能力面板：计划、MCP、技能、权限、知识库切换。

**验证点**：当前 `prototype` 里的关键路径全部接真实 API，用户能用 UI 完成单Agent任务。

## Phase 3 · 任务执行闭环（1-2 周）

- [ ] 对话任务真实执行：AgentScope ReActAgent + SSE 消息流。
- [ ] 计划面板：展示任务步骤、运行状态、失败原因。
- [ ] MCP 面板：展示可用 MCP server，能启用/停用。
- [ ] 技能面板：展示Agent可用技能，先支持本地内置技能。
- [ ] 权限面板：敏感工具调用前询问，记录规则。
- [ ] 知识库面板：展示Agent可用 Hub 知识范围，支持检索来源回链。

**验证点**：用户能在对话页派任务，右侧面板能同步展示计划、工具、权限、知识调用情况。

## Phase 4 · 自动任务与记忆（1 周）

- [ ] 自动任务创建入口和列表。
- [ ] 定时/手动触发自动任务。
- [ ] 工作记录统计：任务数、项目数、最近活动。
- [ ] 记忆 v0：会话摘要、Agent长期记忆、可查看/可清理。
- [ ] Agent主页接真实工作记录和记忆数据。

**验证点**：Agent能执行一次自动任务，并在Agent主页留下可见工作记录和记忆。

## Phase 5 · 打磨与试点（1 周）

- [ ] 角色模板第一批：前端、后端、产品、运营、测试。
- [ ] 安装包、签名、公证、更新通道。
- [ ] 首启引导：连接 Hub、选择工作目录、创建第一个Agent。
- [ ] 试点 2-3 个真实同事，收集任务路径问题。

**验证点**：非开发用户能独立安装、创建Agent、连接 Hub、完成第一条对话任务。

## 持续纪律

- 不把能力配置做成散落的设置页，优先落到右侧任务/能力面板。
- 不提前抽象多Agent协作，先把单Agent生命周期打稳。
- AgentScope 导入收口 adapter。
- SSE/事件契约集中定义。
- Hub 连接封装为 Connector，避免 MCP/HTTP 选择扩散到业务代码。
