# 01 · 产品设计

## 产品定位

**Agent-Smith 是本地数字员工工作台。**

它要解决的问题不是「怎么再做一个聊天机器人」，而是「怎么把 Agent 变成公司同事能理解、能管理、能长期使用的数字员工」。用户看到的核心对象不是模型、prompt 或插件，而是一个个可以入职、配置、分派任务、查看工作记录的员工。

一句话：

> 用户在本机雇佣和管理数字员工，把对话任务、自动任务、本地工具调用和团队知识检索交给它们执行。

## 目标用户

**第一批用户**：坐在电脑前工作的公司同事，尤其是需要频繁处理知识检索、文档整理、代码/运营/产品任务的人。

这些用户不一定理解 AgentScope、MCP、向量检索，但能理解：

- 我需要一个前端工程师、后端工程师、产品经理、运营专员这样的数字员工。
- 这个员工运行在我的电脑上，能访问我允许的工作目录和工具。
- 我可以给它发起对话任务，也可以设置自动任务。
- 我能看到它做了什么、用了什么能力、留下了哪些记忆。

## 当前产品形态

当前原型确定的主形态是「数字员工 + 任务 + 右侧能力面板」。

### 1. 数字员工管理

这是用户进入产品后的默认入口。

- `我的员工 / 我的群组`
- 员工卡片：头像、名称、角色、运行环境、设备、在线状态、简介、知识/能力标签
- 快捷动作：创建对话任务、创建自动任务
- 新建员工入口

这里回答的是：「我有哪些数字员工？它们现在能不能工作？」

### 2. 新建数字员工

用户从模板或自定义角色创建员工。

- 角色模板：前端开发工程师、后端工程师、项目管理员、UI 设计师、测试工程师、产品经理、数据分析师、DevOps 工程师、内容运营专员
- 员工名称、头像、简介
- 自定义角色模板：`identity.md`、`persona.md`、`bible.md`
- 可绑定 Skill Zip、插件市场能力、MCP

这里回答的是：「我要雇佣一个什么角色？它的职责、工作风格和能力是什么？」

### 3. 员工详情首页

点击员工卡片后进入员工主页，而不是进入一个泛工作区。

左侧员工菜单：

- 首页
- 项目
- 自动任务
- 任务
- 记忆
- 技能
- 连接器
- IM
- 权限

首页内容：

- 员工身份卡
- 角色、在线状态、入职时间、简介、编辑入口
- 工作记录：入职天数、自动任务、对话任务、项目数、时间线热力图
- 记忆与积累

这里回答的是：「这个员工是谁？最近做了什么？积累了什么？」

### 4. 对话任务

对话页不是普通聊天页，而是给数字员工派发任务的任务入口。

- 左侧：群组、会话列表
- 中央：员工问候、任务建议、消息输入区
- 顶部：创建对话任务、创建自动任务、任务列表
- 输入区：消息、工作目录上下文、自动模式

这里回答的是：「我要让这个员工马上处理什么？」

### 5. 右侧任务/能力面板

`任务列表` 不跳转到独立工作区，而是在对话页右侧打开停靠面板。

面板可切换：

- 计划
- MCP
- 技能
- 权限
- 知识库

这里回答的是：「当前任务会怎么计划？能用哪些工具？权限和知识边界是什么？」

## 产品支柱

1. **数字员工是第一对象**：所有信息架构围绕员工展开，而不是围绕模型或插件展开。
2. **本地运行是信任基础**：员工、记忆、会话、本地工具权限都在本机；用户知道它在哪台设备上运行。
3. **任务是核心交互**：对话任务和自动任务优先，圆桌/群组协作后置。
4. **能力面板必须可见**：计划、MCP、技能、权限、知识库不能藏在设置里，要在当前任务边上出现。
5. **员工要有履历**：工作记录和记忆积累是数字员工区别于一次性聊天的关键。

## 核心场景故事

### 场景 A：创建一个前端数字员工

1. 用户打开 Agent-Smith，进入「我的数字员工」。
2. 点击「新建员工」。
3. 选择「前端开发工程师」模板。
4. 设置名称、头像、简介。
5. 保存并启用后，员工出现在列表中，状态显示为本地设备在线。

### 场景 B：派发一个对话任务

1. 用户点击员工卡片里的「创建对话任务」。
2. 进入对话页，看到员工问候和任务建议。
3. 用户选择工作目录，输入任务。
4. 点击「任务列表」打开右侧面板。
5. 面板显示计划，并可切到 MCP、技能、权限、知识库查看当前任务能力边界。

### 场景 C：查看员工履历

1. 用户点击员工卡片本体。
2. 进入员工详情首页。
3. 查看员工身份、入职时间、工作记录、任务数量和记忆积累。
4. 如果职责不准确，点击编辑调整员工简介、技能或连接器。

## MVP 边界

### v0 必须做

- Electron 壳 + Python 本地 server 骨架
- 数字员工 CRUD：模板创建、自定义角色模板、员工主页
- 单员工对话任务：会话列表、消息流、任务建议、输入区
- 右侧任务/能力面板：计划、MCP、技能、权限、知识库
- 自动任务的基础模型：创建入口、任务列表、状态展示
- Hub 连接：MCP token / 检索 API 配置，能让员工检索团队知识
- 本地权限：工作目录限制、工具白名单、权限询问

### v0 明确不做

- 不做独立的大「工作区」页面，任务能力在对话页右侧面板里呈现。
- 不把圆桌会议作为首发核心；它是数字员工成熟后的协作增强能力。
- 不做移动端。
- 不做国际化，中文优先。
- 不做本地向量知识库，团队语义检索由 Hub 提供。
- 不复用 QoderWake / openhanako 的代码、品牌、图标或资产。

## 后续能力

- 多员工群组协作
- 圆桌会议 / 多角色方案综合
- 技能市场
- 角色模板 zip 导入导出
- 记忆压缩和长期事实库
- IM 接入
- 更强的自动任务调度和通知
