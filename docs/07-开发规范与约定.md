# 开发规范与约定

本文档定义 Agent-Smith 项目的分层规则、命名约定、代码风格、提交规范和日常开发命令。

---

## 1. 分层规则

核心分层，**单向依赖，不可逆**：

```
shell/ ──HTTP/SSE──→ server/ ──import──→ engine/ ──import──→ common/
                              │                    ↑
                              └── 读取 ──→ agents/  ┘
```

| 层 | 目录 | 职责 | 禁止事项 |
|----|------|------|---------|
| **common/** | 公共基础设施 | 路径、配置合并、SQLite 连接、YAML/JSON 工具 | 零业务流程。不依赖任何其他层 |
| **engine/** | Agent 执行引擎 | LLM 调用、Prompt 组装、工具/技能注册、执行引擎（DAG + ReAct）、记忆、安全 | 零平台逻辑。不知道 FastAPI、HTTP、Agent 管理 |
| **agents/** | Agent 内容定义 | Smith 身份种子、内置技能、内置工具、安全规则 | 零代码依赖（工具 Provider .py 文件除外）。身份/技能文件是纯文本 |
| **server/** | 平台后端 | FastAPI 应用，Agent/会话/任务管理 | 不直接操作 LLM，不包含 Agent 执行逻辑 |
| **shell/** | 终端前端 | Ink/React TUI | 通过 HTTP/SSE 与 server 通信，不直接调用 Python |

### server/ 内部分层

`server/app/` 保持通用 FastAPI application package 习惯，不再和历史 root `app/` 前端混用。

```
server/app/
├── routers/           # 薄壳：参数提取 → 调 service → 返回结果
├── services/          # 编排层：连接 engine + infrastructure，不直接操作数据库
├── infrastructure/    # 持久化：Repository 模式，SQL 写在这里
├── schemas/           # Pydantic 请求/响应模型
├── utils/             # 工具函数
└── main.py            # FastAPI 应用入口
```

**铁律**：
- `routers/` 不做业务判断，只做参数提取 → 调 service → 返回
- `services/` 不直接写 SQL，通过 infrastructure 的 Repository 操作数据
- `infrastructure/` 不包含业务逻辑，只有数据访问

## 2. 命名约定

### 目录和文件

| 类型 | 命名风格 | 示例 |
|------|---------|------|
| Smith 身份目录 | 固定 | `agents/smith/` |
| 技能目录 | kebab-case | `full-stack-product`, `code-review` |
| Python 模块 | snake_case | `employee_service.py`, `skill_chain.py` |
| Python 包 | snake_case | `engine/execution/`, `server/app/routers/` |

### Python 代码

| 类型 | 命名风格 | 示例 |
|------|---------|------|
| Pydantic 模型 | PascalCase + 后缀 | `EmployeeCreate`, `EmployeeUpdate`, `EmployeeOut` |
| 类 | PascalCase | `ToolRegistry`, `SkillStore`, `EmployeeService` |
| 函数/方法 | snake_case | `list_employees()`, `load_providers()` |
| 常量 | UPPER_SNAKE_CASE | `MAX_OUTPUT`, `MAX_FILE_SIZE`, `_MAX_VERSIONS` |
| 私有函数 | `_` 前缀 | `_validate_ref()`, `_check_sensitive()` |

### API 路径

| 规则 | 示例 |
|------|------|
| 单 Smith 主路径 | `/api/agent`, `/api/agent/sessions`, `/api/agent/skills` |
| 操作动词（非 CRUD） | `/api/agent/sessions/{id}/messages` |

### 数据库

| 类型 | 命名风格 | 示例 |
|------|---------|------|
| 表名 | 复数名词, snake_case | `employees`, `sessions`, `auto_tasks` |
| 列名 | snake_case | `created_at`, `employee_id` |

### ID 生成

| 类型 | 格式 | 长度 |
|------|------|------|
| Agent ID | `uuid4().hex[:8]` | 8 字符 |
| 群组 ID | `uuid4().hex[:8]` | 8 字符 |
| 会话 ID | `uuid4().hex[:12]` | 12 字符 |
| 消息 ID | `uuid4().hex[:12]` | 12 字符 |
| 任务 ID | `uuid4().hex[:12]` | 12 字符 |
| 记忆 ID | `uuid4().hex[:12]` | 12 字符 |

## 3. 代码风格

### Python

- **版本**：Python 3.11+
- **异步**：全面使用 `async/await`，数据库操作走 `aiosqlite`
- **类型标注**：`from __future__ import annotations`，使用 `str | None` 语法
- **数据模型**：Pydantic v2（`BaseModel`），字段用 `Field()` 提供默认值和描述
- **HTTP 框架**：FastAPI，路由定义在 `server/app/routers/`
- **数据库**：SQLite（WAL 模式），`common/database.py` 只提供连接；业务 schema 和已初始化连接由 `server/app/infrastructure/` 管理

### Repository 模式

每个数据库表对应一个 Repository 类，放在 `server/app/infrastructure/` 下：

```python
class EmployeeRepo:
    def __init__(self, db): ...
    async def list_all(self) -> list[dict]: ...
    async def get(self, employee_id: str) -> dict | None: ...
    async def create(self, data: dict) -> dict: ...
    async def update(self, employee_id: str, data: dict) -> dict | None: ...
    async def delete(self, employee_id: str) -> bool: ...
```

### 工具 Provider 模式

参见 [06-Agent模板与技能规范.md](06-Agent模板与技能规范.md) 第 5 节。每个工具文件导出 `TOOL_META` 字典和 `async def execute(**kwargs) -> str` 函数。

## 4. 提交规范

遵循 [Conventional Commits](https://www.conventionalcommits.org/)：

```
<type>: <description>

[optional body]
```

### 类型前缀

| 前缀 | 用途 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档变更 |
| `refactor` | 重构（不改变外部行为） |
| `test` | 测试相关 |
| `chore` | 构建/配置/依赖等杂务 |

### 示例

```
feat: 引入统一设计系统 (AppPalette + AppTypography + 卡片样式)
fix: 修复会话消息排序问题
docs: 整体架构文档 + OpenHanako 调研 + TODO 待优化清单
refactor: macOS UI 重构 + 清理临时资源
```

## 5. 新增能力操作

Agent-Smith 不再通过新增 Agent 身份扩展能力。新增具体能力时，在 `agents/skills/` 下创建 skill：

```bash
mkdir agents/skills/<skill-name>/
# 填写：
# - SKILL.md       (必填) frontmatter + 具体工作流/SOP/领域知识
```

Smith 的基础身份只在 `agents/smith/` 维护；不要新增其他模板目录。

## 6. 开发命令

### 后端启动

```bash
# 标准启动（开发模式，热重载）
cd server && uv run uvicorn app.main:app --port 8140 --reload

# 带 LLM 配置启动
AGENTSMITH_LLM_API_KEY="sk-xxx" \
AGENTSMITH_LLM_BASE_URL="https://..." \
AGENTSMITH_LLM_MODEL="glm-4.7" \
uv run uvicorn app.main:app --port 8140
```

### 依赖安装

```bash
cd server && uv sync    # 后端依赖
cd engine && uv sync    # 引擎依赖
cd common && uv sync    # 公共依赖
```

### 前端构建

```bash
cd shell && npm ci && npm run build
```

## 8. 数据目录

运行时数据存储在用户 Home 目录下：

```
~/.agent-smith/
├── config.yaml                    # 平台级配置（LLM key/url/model）
├── employees/<id>/                # Agent 实例（从模板复制 + 运行时积累）
│   ├── role.md                    # 从模板复制
│   ├── style.md
│   ├── workflow.md
│   ├── toolbox.md
│   ├── context.md                 # 运行时由 UserPreferenceLearner 填充
│   ├── config.yaml
│   ├── memory/                    # 记忆（.md 文件 + recent.jsonl）
│   ├── skills/                    # Agent 自装/自进化技能
│   │   └── <name>/
│   │       ├── SKILL.md
│   │       └── .versions/         # 版本快照（最多 10 个）
│   └── sessions/                  # 会话记录
└── sqlite/agent-smith.sqlite      # 索引库（WAL 模式）
```

**关键区分**：
- `agents/smith/` — Smith 的只读内置身份种子，Git 跟踪
- `agents/skills/` — 能力扩展入口，按任务按需加载
- `~/.agent-smith/employees/<id>/` — legacy 兼容路径，语义上仍是 Smith profile
- 运行时读写的是 `~/.agent-smith/` 下的副本，**不碰模板原件**
