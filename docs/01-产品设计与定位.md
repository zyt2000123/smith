# 01 · 产品设计与定位

## 产品定位

**Agent-Smith 是常驻个人电脑的个人助手 Agent——越用越聪明。**

它要解决的问题不是「怎么再做一个聊天机器人」，而是「怎么让一个本地 Agent 成为可长期协作的个人助手」。用户看到的核心对象不是模型、prompt 或插件，而是一个持续存在、能执行、能积累、能被管理的 Agent 实例。它运行在本机环境中，围绕用户的文件、任务、知识、技能、权限和工作记录持续工作。

> 过渡说明：当前实现中的 server / 数据目录 / SwiftUI 文件结构仍保留 `employee` / `employees` 命名，作为纯实现层的兼容壳；产品语义、界面文案、默认模板和 CLI 均已收口为单一“个人助手 Agent”。

一句话：

> 用户在本机拥有并持续使用一个个人助手 Agent，把对话任务、自动任务、本地工具调用和团队知识检索交给它执行。

## 目标用户

**第一批用户**：坐在电脑前工作的知识工作者，尤其是需要频繁处理知识检索、文档整理、代码、运营、产品和调研任务的人。

这些用户不一定理解 Agent 框架、MCP、向量检索，但能理解：

- 我需要一个常驻本机、懂我当前工作上下文的个人助手。
- 这个助手运行在我的电脑上，能访问我允许的工作目录和工具。
- 我可以给它发起对话任务，也可以设置自动任务。
- 我能看到它做了什么、用了什么能力、留下了哪些记忆。

## 当前产品形态

当前产品确定的主形态是「个人助手 Agent + 任务 + 右侧能力面板」。

### 1. 个人助手入口

用户进入产品后的默认入口，以“我的 Agent / 我的群组”呈现，首先承载的是默认个人助手 Agent 的入口与状态。

- `我的 Agent / 我的群组` 切换
- Agent 卡片列表：头像、名称、角色、运行设备、在线状态、简介
- 快捷动作：创建对话任务、创建自动任务
- 新建模板化 Agent 入口

这里回答的是：「我的个人助手现在能不能工作？如果我需要，也能不能从模板扩展出额外专用 Agent？」

### 2. 新建 Agent 模板实例

用户从角色模板创建 Agent 实例。当前默认内置的主模板是 `personal-assistant`，同时仍保留若干专用模板以支撑后续扩展和兼容现有界面。

| 模板 | 目录名 |
|---|---|
| 个人助手 | `personal-assistant` |
| 前端工程师 | `frontend-engineer` |
| 后端工程师 | `backend-engineer` |
| 产品经理 | `product-manager` |
| UI 设计师 | `ui-designer` |
| 测试工程师 | `test-engineer` |
| 运维工程师 | `devops-engineer` |
| 数据分析师 | `data-analyst` |
| 内容运营 | `content-ops` |

创建时设置名称、头像、简介，从模板复制到本地运行目录。对当前产品来说，最重要的是先把默认个人助手跑起来；多角色模板属于能力扩展，而不是主叙事。

这里回答的是：「我要先用哪个 Agent 作为我的默认助手？如果需要再扩展，它的职责、工作风格和能力是什么？」

### 3. 助手详情首页

点击卡片后进入助手主页。左侧提供 9 个标签页导航：

| 标签 | 内容 |
|---|---|
| 首页 | Agent 身份卡、工作记录、记忆与积累 |
| 项目 | Agent 参与的项目 |
| 自动任务 | 定时/手动触发的自动化任务 |
| 任务 | 对话任务列表 |
| 记忆 | Agent 积累的记忆条目 |
| 技能 | 内置技能 + 自装技能 |
| 连接器 | MCP、Hub API 等外部连接 |
| IM | 即时通讯接入（预留） |
| 权限 | 工作目录、工具白名单、操作授权 |

首页内容：

- **Agent 身份卡** — 角色、在线状态、启用时间、简介、编辑入口
- **工作记录** — 入职天数、自动任务数、对话任务数、项目数；GitHub 风格的时间线热力图
- **记忆与积累** — 最近记忆条目、知识/能力标签

这里回答的是：「这个助手是谁？最近做了什么？积累了什么？」

### 4. 对话任务

对话页不是普通聊天页，而是给个人助手 Agent 派发任务的任务入口。

- **左侧**：群组、会话列表
- **中央**：Agent 问候、任务建议、消息输入区（支持工作目录上下文、自动模式）
- **右侧能力面板**：停靠在对话页右侧

消息流中展示 Agent 的思考过程、工具调用、输出结果和来源回链。

这里回答的是：「我要让这个助手马上处理什么？」

### 5. 右侧任务/能力面板

`任务列表` 不跳转到独立工作区，而是在对话页右侧打开停靠面板。面板可切换：

| 面板 | 展示内容 |
|---|---|
| 计划 | 当前任务的技能链编排、执行进度 |
| MCP | 已连接的 MCP server 列表 |
| 技能 | 当前可用的内置技能 + 自装技能 |
| 权限 | 工作目录限制、工具白名单、操作授权状态 |
| 知识库 | Hub 知识范围、检索能力边界 |

这里回答的是：「当前任务会怎么计划？能用哪些工具？权限和知识边界是什么？」

## 产品支柱

1. **个人助手 Agent 是第一对象** — 所有信息架构围绕长期协作的 Agent 展开，而不是围绕模型或插件展开。
2. **本地运行是信任基础** — Agent、记忆、会话、本地工具权限都在本机；用户知道它在哪台设备上运行。
3. **任务是核心交互** — 对话任务和自动任务优先，圆桌/群组协作后置。
4. **能力面板必须可见** — 计划、MCP、技能、权限、知识库不能藏在设置里，要在当前任务边上出现。
5. **助手要有连续性** — 工作记录和记忆积累是个人助手区别于一次性聊天的关键。

## 核心场景故事

### 场景 A：初始化默认个人助手

1. 用户打开 Agent-Smith，进入默认入口。
2. 系统确保一个内置 `personal-assistant` 模板可用。
3. 用户设置名称、头像、简介或直接使用默认助手。
4. 保存后，系统从 `agents/templates/personal-assistant/` 复制 9 个文件到 `~/.agent-smith/employees/<id>/`。
5. Agent 出现在列表中，状态显示为本地设备在线。

### 场景 B：派发一个对话任务

1. 用户点击 Agent 卡片里的「创建对话任务」。
2. 进入对话页，看到 Agent 问候和任务建议。
3. 用户选择工作目录，输入任务（如「帮我修复登录页的表单验证 bug」）。
4. 引擎自动路由到 Bug Fix 技能链：sde-debug → planning → testing-strategy → change-validation → code-review。
5. 右侧面板实时展示执行计划和进度。
6. Agent 在 ReAct 循环中调用工具完成修复，消息流中展示过程和结果。

### 场景 C：查看助手履历

1. 用户点击 Agent 卡片本体。
2. 进入详情首页。
3. 查看 Agent 身份、启用时间、工作记录、任务数量和记忆积累。
4. 如果职责不准确，点击编辑调整简介、技能或连接器。

## MVP 边界

### v0 必须做

- SwiftUI macOS 原生前端 + Python 本地 server 骨架
- 自研 Agent 引擎（混合 DAG + ReAct 架构）
- 默认个人助手 Agent：模板创建、实例初始化、详情主页
- 单 Agent 对话任务：会话列表、消息流、任务建议、输入区
- 右侧任务/能力面板：计划、MCP、技能、权限、知识库
- 自动任务：创建入口、任务列表、cron/interval 触发、状态展示
- Hub 连接：MCP token / 检索 API 配置，能让 Agent 检索团队知识
- 本地权限：工作目录限制、工具白名单、24 条安全规则拦截

### v0 明确不做

- 不做独立的大「工作区」页面，任务能力在对话页右侧面板里呈现
- 不把圆桌会议作为首发核心；它是个人助手成熟后的协作增强能力
- 不做移动端
- 不做国际化，中文优先
- 不做本地向量知识库，团队语义检索由 Hub 提供
- 不复用 QoderWake / OpenHanako 的代码、品牌、图标或资产

### 技术选型说明

| 维度 | 最终选型 | 淘汰方案 |
|---|---|---|
| 前端 | SwiftUI macOS 原生（Swift 6.1, macOS 15+, SPM, 零第三方依赖） | ~~Electron~~、~~React + Semi Design~~ |
| Agent 引擎 | 自研（混合 DAG + ReAct, engine/ 目录） | ~~AgentScope~~、~~LangChain~~ |
| 后端 | Python + FastAPI | 不变 |
| 存储 | 文件系统 + SQLite WAL | 不变 |

## 与 AI Knowledge Hub 的边界

| | Agent-Smith（本地） | AI Knowledge Hub（云端） |
|---|---|---|
| 核心对象 | 个人助手 Agent、任务、技能、连接器、权限、记忆 | 知识库、文档、Wiki、用户、权限 |
| 主要用户动作 | 初始化 Agent、派任务、看工作记录、配置能力 | 上传文档、管理知识库、检索和浏览 Wiki |
| 数据归属 | 个人电脑（`~/.agent-smith/`） | 团队服务器 |
| 运行环境 | 本地桌面应用 | 云端 Web 服务 |
| 对外能力 | 调用 MCP / 本地工具 | 提供 MCP server / 检索 API |

**知识是团队的，助手是个人的。** Hub 对 Agent-Smith 来说是一组可调用的团队知识工具。Hub 不可用时，本地 Agent 仍可工作，但知识检索能力降级。
