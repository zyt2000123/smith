AGENT-SMITH / MEMORY SYSTEM DESIGN

Memory 不是日志。
它是一条被治理的事实生命周期。

让单常驻 Agent 记得住、想得起、忘得掉、学得会，同时保证记忆故障不阻塞主流程。

THE THESIS
每一条记忆都要回答两个问题：

“你凭什么记得？”以及“现在为什么要想起？”

WRITE / 门槛

先判断：这次对话值得记吗？

只有 had_tools=true 的对话才落入 recent.jsonl，并同步创建一条带 Evidence 的 Markdown 条目。纯闲聊被挡在门外，避免噪音稀释检索质量。

02 / CONSOLIDATION

时间分层，让短期噪音不挤掉长期事实

四层不是四份摘要的堆叠，而是四种不同的“保留理由”：今天有用、这周反复、长期稳定、用户事实。

today.md
日内重建

把当天 3–5 个关键事件压成短期上下文，允许快速过期。

3–5 events / ≤400 chars
week.md
7 日滚动窗口

提炼重复出现的主题，观察一次性事件是否正在变成模式。

3–5 themes / ≤400 chars
longterm.md
折叠式累积

把新周摘要 merge 进长期记忆，LLM 负责去重与折叠。

carry + merge / ≤600 chars
facts.md
30 日耐久事实

只留下关于用户的稳定事实，回答“这个人总体是谁”。

carry forward / ≤500 chars
03 / SOURCE OF TRUTH

可读的文件，才是用户真正拥有的记忆

SQLite 只是派生索引，删掉也能回到关键词扫描。事实源必须能被打开、编辑、迁移和删除。

明文事实源

可审计 / 可编辑
memory/
├── agent/ · 个人经验
├── project/ · 跨 Agent 共识
├── recent.jsonl · 编译原料
└── facts.md · 耐久事实
每个 *.md 都带 id / scope / evidence

`Evidence` 是一等公民：记忆不只说“是什么”，还要说“凭什么”。

可丢弃派生物

快 / 可降级
search.sqlite
├── memory_fts · FTS5 / BM25
├── memory_vec · sqlite-vec
└── RRF(k=60) · rank fusion
向量不可用 → 纯 FTS5 → 全文件扫描

索引追求“想得起”，不承担“记得是什么”的最终权威。

04 / DREAM

记忆也会腐化，所以要定期做梦

每 5 次对话触发一次整理。先看计划，再执行破坏性操作，为未来人工确认保留边界。

清秘

命中 API key、私钥或 token 的条目直接进入 secret_ids，避免每次注入都重复泄漏。

剪枝

last_accessed 超过 30 天的条目被视为低价值，释放 prompt 与存储空间。

去重合并

关键词重叠 ≥70% 时保留最新条目，并把旧条目独有信息并入其中。

模式提取

同主题条目 ≥3 条时，生成一条更高层的 Pattern，完成从观察到反思。

SAFETY BOUNDARY

Dream 不是静默删除器

整理被拆成只读 preview 与执行 apply 两步：先生成计划，再决定是否接受。

05 / RECALL

双轨读取：画像 + 情景，缺一不可

被动注入保证 Agent 有稳定画像；主动检索让它在需要时召回一条具体经验。两个轨道回答不同问题。

06 / DECISIONS

真正重要的是：为什么这样设计？

这些不是实现偏好，而是围绕“本地、单 Agent、可审计、低部署成本”的明确取舍。

DR-01 / STORAGE

明文 Markdown，而不是黑盒数据库

记忆是用户资产。文件天然可查看、可编辑、可 Git、可迁移；SQLite 只做可重建的速度层。

代价：文件写入不是事务；单用户桌面场景接受。
DR-02 / COMPRESSION

时间分层，而不是一份滚动总摘要

一次性事件不会冲刷稳定偏好；today/week/longterm/facts 让不同时间尺度各自归位。

代价：四层编译有 LLM 成本；指纹缓存 + 每 5 次触发缓解。
DR-03 / RECALL

画像摘要 + query-time 双轨

只有摘要会与当前任务脱节；只有 RAG 又无法回答“这个用户总体是谁”。

代价：每次组 prompt 多一次轻量检索；失败静默降级。
THE CONTRACT

记忆是增强，不是身份。

当预算不足时，宁可失忆，不可失格。Memory 负责让 Agent 更连续、更懂用户，但任何读写、索引、编译或 Dream 故障，都不能夺走主流程的可用性。
